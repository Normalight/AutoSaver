# AutoSaver 界面一体化重设计规格

日期：2026-05-06

## 背景

当前主窗口使用表格式 `ListView/GridView`，在宽窗口下只有少数列占用左侧，右侧出现大片空白。设置页、滚动条、主题资源和窗口顶部控件之间也存在视觉不一致或遮挡问题。发布流程中 `CHANGELOG.md` 作为整体内容使用，导致没有按当前版本提取对应章节。

## 目标

- 将界面整体改为现代卡片仪表盘风格。
- 主窗口使用自定义无边框标题栏，重写关闭、最小化、最大化按钮。
- 程序列表改为固定单列卡片，避免表格列后方空白。
- 重写按钮、选择框、输入框、复选框、滚动条和卡片样式。
- 深色、浅色、跟随系统主题切换后，所有控件和窗口区域保持一致。
- 检查间隔默认值为 30 秒，支持时、分、秒单位选择，默认单位为秒。
- `CHANGELOG.md` 展示或发布内容按当前版本号提取对应章节。
- 窗口和设置页支持自适应缩放，不遮挡内容。

## 非目标

- 不改变自动保存、进程检测、托盘菜单、通知弹窗的核心业务逻辑。
- 不引入新的 NuGet 依赖。
- 不迁移现有 INI 配置存储。

## 总体架构

### 主窗口

主窗口拆成四个区域：

1. 自定义标题栏
   - 左侧显示 `AutoSaver` 和 `v{App.Version}`。
   - 右侧显示最小化、最大化/还原、关闭按钮。
   - 标题栏承担拖拽移动窗口的行为。

2. 快捷操作卡片
   - 显示应用说明和主要操作。
   - 包含“添加程序”主按钮和“设置”次按钮。

3. 程序列表卡片区
   - 使用固定单列卡片列表。
   - 每个程序一张卡片，展示程序名、exe 摘要、运行状态、保存间隔、最近保存信息和操作按钮。
   - 列表区域使用统一滚动条样式，滚动条不覆盖内容。

4. 底部状态条
   - 显示最近保存信息或当前运行摘要。
   - 使用轻量信息条，不占用过多高度。

### 设置窗口

设置窗口采用同一套卡片视觉语言：

- 主题设置。
- 检查间隔设置。
- 开机自启。
- 关闭到托盘。
- 保存通知。

每个设置组之间使用统一分割线。设置窗口应可在小尺寸下滚动，不允许内容被底部按钮或窗口边界遮挡。

## 样式系统

主题资源继续位于 `Themes/LightTheme.xaml` 和 `Themes/DarkTheme.xaml`，但需要补齐统一资源：

- 背景：窗口背景、卡片背景、悬浮背景。
- 文本：主文本、次文本、弱化文本。
- 边框和分割线。
- 阴影或等价层次效果。
- 主按钮、次按钮、幽灵按钮、危险按钮。
- 输入框、选择框、复选框。
- 滚动条。
- 程序卡片和状态徽章。
- 窗口控制按钮。

所有控件必须具有 hover、pressed、disabled 的一致状态。深色和浅色主题必须提供相同 key，避免切换后局部资源缺失。

## 检查间隔配置

设置页中的检查间隔由数值输入框和单位选择框组成：

- 默认数值：30。
- 默认单位：秒。
- 可选单位：秒、分、时。
- 保存时统一换算为秒，供现有监控定时器继续使用。
- 读取旧配置时，如果只有 `check_interval_sec`，应根据秒数展示为合适的数值和单位，不能破坏现有配置。

## 版本号和 Changelog

版本号继续由 `VERSION` 文件提供，主窗口显示 `v{App.Version}`。

Changelog 处理改为按版本提取：

- 从 `CHANGELOG.md` 中匹配 `## [x.y.z] - date` 章节。
- 当前版本来自 `VERSION`。
- 输出内容只包含当前版本章节，不包含其它版本。
- 如果没有找到对应章节，使用简短回退内容，说明未找到当前版本更新记录。

发布流程也应使用该提取结果，而不是直接把整份 `CHANGELOG.md` 作为 release body。

## 数据流

- 启动时读取 `VERSION`，设置 `App.Version`。
- 初始化主题时根据配置和系统设置加载对应资源字典。
- 打开主窗口时将程序配置映射到卡片展示模型。
- 程序状态和最近保存信息更新时刷新对应卡片内容和底部状态条。
- 保存设置时写入 INI，并立即应用主题与检查间隔。

## 错误处理

- 版本文件读取失败时继续使用内置默认版本。
- Changelog 对应章节缺失时显示回退文案，不影响应用启动。
- 检查间隔输入非法时阻止保存并提示用户。
- 主题切换只替换主题资源字典，不移除其它全局资源。

## 测试方案

需要验证：

- 深色、浅色、跟随系统主题切换后，主窗口、设置窗口、按钮、输入框、选择框、列表卡片和滚动条全部同步变化。
- 主窗口标题栏按钮可最小化、最大化/还原、关闭窗口，标题栏可拖拽。
- 程序列表为固定单列卡片，不再出现表格列后方大片空白。
- 设置窗口内容不会被覆盖，小尺寸下可以滚动。
- 检查间隔默认显示 30 秒，保存后内部仍以秒数驱动监控。
- 版本号显示与 `VERSION` 一致。
- Changelog 只提取当前版本章节。
- 窗口缩放到最小尺寸时，各区域仍可读且无重叠。
